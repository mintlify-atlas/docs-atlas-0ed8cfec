## Overview

The **Individual Seller View** (Vista por Vendedor) provides each sales representative with their own personalized dashboard. Sellers can access their performance metrics, see their position in the team ranking, track their personal evolution, and manage their customer and product portfolio—all without waiting for external reports.

## Key Features

### Seller Profile Card

Each seller has a personalized header showing:
- Full name and profile badge
- Geographic zone assignment
- Years of experience (seniority)
- Performance profile (Star/Stable/Development/Problematic)

The profile is color-coded based on performance classification:
- **Green**: Star performer
- **Blue**: Stable performer
- **Yellow**: Development profile
- **Red**: Requires improvement

### Personal KPIs

Four main metrics track individual performance:

**Sales for the Period**
- Total net sales
- Percentage variation vs. previous period
- Progress toward personal objective
- Compliance percentage

**Quota Compliance**
- Achievement percentage
- Visual indicator (green/yellow/red)
- Goal comparison

**Customers Visited**
- Number of unique customers served
- Coverage metric

**Average Ticket**
- Mean transaction value
- Quality of sales indicator

<Info>
All personal KPIs are automatically filtered to show only the selected seller's data, ensuring privacy and focus.
</Info>

## Implementation Details

The Individual Seller View is implemented in `visor_kpi/pages/2_vendedores.py`.

### Seller Selection

The page provides a dropdown to select any active seller:

```python
vend_activos = vend_df[vend_df["activo"] == True].copy()
opciones_v   = {f"{r['nombre_completo']} ({r['zona']})": r["id_vendedor"]
                for _, r in vend_activos.iterrows()}

selected_label = st.selectbox(
    "👤 Seleccionar vendedor",
    options = list(opciones_v.keys()),
    key     = "_sel_vendedor_page",
)
vendedor_id   = opciones_v[selected_label]
vendedor_info = vend_activos[vend_activos["id_vendedor"] == vendedor_id].iloc[0]
```

### Data Loading for Selected Seller

```python
# Load data filtered for the selected seller
df     = get_filtered_data(fecha_desde, fecha_hasta, 
                           [vendedor_id], zonas_sel, canales_sel)
df_ant = get_filtered_data_periodo_anterior(fecha_desde, fecha_hasta, 
                                             [vendedor_id], zonas_sel, canales_sel)

df_eq  = get_filtered_data(fecha_desde, fecha_hasta,

df_eq_ant = get_filtered_data_periodo_anterior(fecha_desde, fecha_hasta, 
                                                None, zonas_sel, canales_sel)
```

## Visualizations

### Personal Compliance Gauge

A visual gauge shows individual quota achievement:

```python
fig_gauge = chart_gauge_cumplimiento(
    valor = kv.get("cumplimiento_pct") or 0,
    label = "Mi Cumplimiento",
)
st.plotly_chart(fig_gauge, use_container_width=True, 
                config={"displayModeBar": False})
```

**Gauge zones:**
- **Green (>90%)**: Exceeding expectations
- **Yellow (70-90%)**: Meeting targets
- **Red (less than 70%)**: Below threshold

### Personal Evolution Chart

Tracks the seller's performance over the last 6 months:

```python
fig_evo = chart_evolucion_ventas(
    evo,
    mostrar_objetivo=True,
    title=f"Evolución personal — últimos 6 meses",

st.plotly_chart(fig_evo, use_container_width=True, 
                config={"displayModeBar": False})
```

**Features:**
- Monthly sales line with trend
- Objective overlay
- Interactive hover tooltips

<Note>
The evolution chart is limited to the last 6 months to keep the view focused and actionable. For longer historical analysis, use the export functionality.
</Note>

### At-Risk Customer Detection

A dedicated section highlights customers who may be churning:

```python
en_riesgo = kcli[kcli["flag_en_riesgo"] == True].copy()
if en_riesgo.empty:
    st.success("✅ No tenés clientes en riesgo en este período.")
else:
    render_ranking_clientes(en_riesgo, top_n=20)
```

**Risk criteria:**
- No purchase in the last 30+ days
- Declining purchase frequency
- Significant drop in order amounts

<Warning>
At-risk customers should be contacted proactively to prevent churn. The system suggests prioritizing category A and B customers who show risk signals.
</Warning>

## Privacy and Access Control

While the current implementation allows any user to select any seller (useful for managers), the architecture supports role-based access control:

```python
# Future implementation example
if user_role == "seller":
    vendedor_id = st.session_state.user_id
    # Lock to logged-in seller only
else:  # manager or admin
    # Show selector for all sellers
    vendedor_id = selected_from_dropdown
```

<Info>
For production deployment, integrate with your authentication system to automatically filter the view to the logged-in seller's data.
</Info>

## Performance Optimization

The page loads data efficiently by:
1. Filtering transactions to the selected seller only
2. Loading team data separately for ranking context
3. Limiting evolution charts to 6 months
4. Using cached data loading functions
5. Rendering only top N items in rankings
