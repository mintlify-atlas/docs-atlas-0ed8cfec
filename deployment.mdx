## Deployment to Streamlit Cloud

Streamlit Cloud offers the easiest way to deploy your Streamlit applications. It provides free hosting with automatic updates from your GitHub repository.

### Prerequisites

1. **GitHub Account**: Your code must be in a GitHub repository
2. **Streamlit Cloud Account**: Sign up at [share.streamlit.io](https://share.streamlit.io)
3. **Mock Data**: Ensure mock data is generated or included in your repository

### Step-by-Step Deployment

#### 1. Prepare Your Repository

Ensure your repository contains:

```
visor_kpi/
├── app.py                    # Main entry point (required)
├── requirements.txt          # Dependencies (required)
├── config.py
├── components/
├── src/
├── pages/
├── assets/
└── data/
    └── raw/
        └── mock_data.xlsx    # Include generated mock data
```

**Important**: Generate and commit the mock data file:

```bash
python data/mock/generate_mock_data.py
git add data/raw/mock_data.xlsx
git commit -m "Add mock data for deployment"
git push
```

#### 2. Deploy on Streamlit Cloud

1. Go to [share.streamlit.io](https://share.streamlit.io)
2. Click **"New app"**
3. Select your repository, branch, and main file:
   - **Repository**: `your-username/visor_kpi`
   - **Branch**: `main` (or your default branch)
   - **Main file path**: `app.py`
4. Click **"Deploy"**

Streamlit Cloud will automatically:
- Install dependencies from `requirements.txt`
- Run your application
- Provide a public URL (e.g., `your-app-name.streamlit.app`)

#### 3. Configure Advanced Settings (Optional)

Click **"Advanced settings"** before deploying to:

- Set **Python version**: `3.9` or higher
- Add **Secrets**: For production data sources (see [Configuration](#configuration-for-production))
- Set **Custom domain**: Use your own domain name (requires paid plan)

### Automatic Updates

Streamlit Cloud automatically redeploys when you push to your repository:

```bash
git add .
git commit -m "Update KPI calculations"
git push
```

Your app will rebuild within a few minutes.


## Configuration Requirements

### Mock Data Mode (Default)

The application works out-of-the-box with mock data:

- Data file: `data/raw/mock_data.xlsx`
- Generated by: `data/mock/generate_mock_data.py`
- Period: 2023-01-01 to 2026-03-31
- Includes: 12 vendors, 180 customers, 60 products, 8,000-40,000 transactions

### Configuration for Production

To use real data in production:

#### Option 1: Replace Mock Data File

1. Prepare your Excel file with the same structure:
   - **Ventas** sheet: id_venta, fecha, id_vendedor, id_cliente, id_producto, cantidad, importe_neto, importe_bruto
   - **Vendedores** sheet: id_vendedor, nombre, zona
   - **Clientes** sheet: id_cliente, razon_social, canal, zona, id_vendedor_asignado, activo
   - **Productos** sheet: id_producto, descripcion, precio_unitario, costo_unitario, activo
   - **Objetivos** sheet: id_vendedor, periodo, objetivo

2. Replace `data/raw/mock_data.xlsx` with your file

3. Update `config.py` if needed:
   ```python
   DATA_PATH = os.path.join(BASE_DIR, "data", "raw", "your_data.xlsx")
   ```

#### Option 2: Use Streamlit Secrets for Database Connection

For connecting to databases (PostgreSQL, MySQL, etc.):

1. Add `.streamlit/secrets.toml` locally (git-ignored):
   ```toml
   [database]
   host = "your-database-host"
   port = 5432
   database = "sales_db"
   user = "your-username"
   password = "your-password"
   ```

2. Add secrets in Streamlit Cloud:
   - Go to your app settings
   - Click **"Secrets"**
   - Paste your secrets in TOML format

3. Modify `src/data_loader.py` to read from database:
   ```python
   import streamlit as st
   import psycopg2
   
   def load_data_from_db():
       conn = psycopg2.connect(
           host=st.secrets["database"]["host"],
           port=st.secrets["database"]["port"],
           database=st.secrets["database"]["database"],
           user=st.secrets["database"]["user"],
           password=st.secrets["database"]["password"],
       )
       ventas = pd.read_sql("SELECT * FROM ventas", conn)
       # ... load other tables
       return {"ventas": ventas, ...}
   ```

### Environment Variables

For non-sensitive configuration:

```python
import os

# In config.py
DEBUG_MODE = os.getenv("DEBUG_MODE", "false").lower() == "true"
DATA_SOURCE = os.getenv("DATA_SOURCE", "mock")  # "mock" or "database"
```

Set in Streamlit Cloud under **Advanced settings** → **Environment variables**.

## Alternative Deployment Options

### Docker Deployment

Create a `Dockerfile`:

```dockerfile
FROM python:3.9-slim

WORKDIR /app

COPY requirements.txt .
RUN pip install --no-cache-dir -r requirements.txt

COPY . .

RUN python data/mock/generate_mock_data.py

EXPOSE 8501

CMD ["streamlit", "run", "app.py", "--server.port=8501", "--server.address=0.0.0.0"]
```

Build and run:

```bash
docker build -t visor-kpi .
docker run -p 8501:8501 visor-kpi
```

### Heroku Deployment

1. Create `setup.sh`:
   ```bash
   mkdir -p ~/.streamlit/
   echo "[server]" > ~/.streamlit/config.toml
   echo "headless = true" >> ~/.streamlit/config.toml
   echo "port = $PORT" >> ~/.streamlit/config.toml
   ```

2. Create `Procfile`:
   ```
   web: sh setup.sh && streamlit run app.py
   ```

3. Deploy:
   ```bash
   heroku create your-app-name
   git push heroku main
   ```

### AWS EC2 Deployment

1. Launch an EC2 instance (Ubuntu)
2. Install dependencies:
   ```bash
   sudo apt update
   sudo apt install python3-pip
   pip3 install -r requirements.txt
   ```
3. Generate mock data and run:
   ```bash
   python3 data/mock/generate_mock_data.py
   streamlit run app.py --server.port 80
   ```
4. Configure security group to allow HTTP traffic (port 80)

## Performance Optimization

### Caching

The application uses Streamlit's caching to optimize performance:

```python
@st.cache_data(ttl=600)  # Cache for 10 minutes
def load_data():
    xl = pd.ExcelFile(DATA_PATH)
    return {
        "ventas": xl.parse("Ventas"),
        # ...
    }
```

### Memory Management

For large datasets:

1. **Increase memory limit** in Streamlit Cloud (paid plans only)
2. **Use chunking** for large Excel files:
   ```python
   chunks = pd.read_excel(DATA_PATH, sheet_name="Ventas", chunksize=10000)
   df = pd.concat(chunks)
   ```
3. **Optimize data types**:
   ```python
   df["id_vendedor"] = df["id_vendedor"].astype("category")
   df["fecha"] = pd.to_datetime(df["fecha"])
   ```

## Monitoring & Troubleshooting

### View Logs in Streamlit Cloud

1. Go to your app dashboard
2. Click **"Manage app"**
3. View **"Logs"** tab for errors and warnings

### Common Issues

#### "File not found" Error

**Solution**: Ensure `mock_data.xlsx` is committed to your repository:
```bash
git add data/raw/mock_data.xlsx
git commit -m "Add mock data"
git push
```

#### "Module not found" Error

**Solution**: Check `requirements.txt` includes all dependencies and versions are compatible.

#### Memory Limit Exceeded

**Solution**: 
- Optimize data loading (see [Performance Optimization](#performance-optimization))
- Upgrade to Streamlit Cloud paid plan for higher limits
- Use database connection instead of Excel files

#### Slow Performance

**Solution**:
- Verify caching is working (`@st.cache_data` decorators)
- Reduce data volume in filters
- Profile with `streamlit run app.py --logger.level=debug`

## Security Best Practices

1. **Never commit secrets**: Use `.gitignore` for `.streamlit/secrets.toml`
2. **Use Streamlit Secrets**: Store credentials in Streamlit Cloud secrets manager
3. **Restrict access**: Use Streamlit Cloud authentication (paid plans)
4. **Validate inputs**: Sanitize user inputs in filters
5. **Use HTTPS**: Streamlit Cloud provides HTTPS by default
