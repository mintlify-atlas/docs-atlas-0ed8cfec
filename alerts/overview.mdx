## Purpose

The Alerts System (`alerts_engine.py`) is a **real-time monitoring engine** that continuously evaluates sales data against 7 business rules to detect:

- **Critical performance issues** requiring immediate attention
- **Business risks** like customer churn or product stagnation
- **Commercial opportunities** for growth and optimization

## How It Works

The alert engine processes transactional data through a rule-based evaluation system:

```python
def get_alertas_activas(
    df: pd.DataFrame,
    df_anterior: pd.DataFrame | None,
    clientes_df: pd.DataFrame,
    productos_df: pd.DataFrame,
    objetivos_df: pd.DataFrame | None = None,
    fecha_hasta: date | None = None,
) -> pd.DataFrame:
    """
    Evaluates all rules and returns DataFrame of active alerts.
    """
```

### Input Data Sources

| Parameter | Description |
|-----------|-------------|
| `df` | Current period sales transactions |
| `df_anterior` | Previous period data (for comparison rules) |
| `clientes_df` | Customer master data with categories |
| `productos_df` | Product catalog |
| `objetivos_df` | Sales targets by salesperson |
| `fecha_hasta` | Evaluation cutoff date (defaults to today) |

### Output Alert Structure

Each alert contains:

- **id_alerta**: Rule identifier (ALT001-ALT007)
- **nombre**: Alert name
- **severidad**: Severity level (alta, media, baja, info)
- **entidad_tipo**: Entity type (vendedor, cliente, producto, equipo)
- **entidad_id** / **entidad_nombre**: Affected entity details
- **valor_actual** / **valor_referencia**: Current vs. threshold values
- **descripcion_detalle**: Specific context for this alert
- **fecha_deteccion**: Detection date
- **accion_sugerida**: Recommended action

## How Alerts Are Triggered

### Evaluation Process

1. **Data Aggregation**: The engine groups transactional data by relevant dimensions (salesperson, customer, product)
2. **Rule Checking**: Each rule's condition is evaluated against aggregated metrics
3. **Alert Creation**: When thresholds are breached, an alert record is created with context
4. **Prioritization**: Alerts are sorted by severity (alta → media → baja → info)

## Display and Prioritization

### Severity-Based Sorting

Alerts are automatically ordered by severity:

```python
orden_sev = {"alta": 0, "media": 1, "baja": 2, "info": 3}
df_alertas["_ord"] = df_alertas["severidad"].map(orden_sev)
df_alertas = df_alertas.sort_values("_ord")
```

This ensures **critical issues appear first** in dashboards and reports.

### Dashboard Integration

The alert DataFrame integrates with:

- **Streamlit dashboards** for visual display with color-coded severity badges
- **Email notifications** for critical alerts (alta severity)
- **Executive reports** summarizing alert counts by type and severity
