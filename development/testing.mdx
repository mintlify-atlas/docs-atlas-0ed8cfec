## Overview

Visor KPI Comercial includes a comprehensive test suite with **51 automated tests** covering data validation, KPI calculations, Pareto analysis, and alert detection. Tests are written using **pytest** and validate both mock data quality and business logic correctness.

## Running Tests

### Prerequisites

Before running tests, generate the mock data:

```bash
python data/mock/generate_mock_data.py
```

### Run All Tests

Execute all tests from the project root:

```bash
pytest
```

### Run Specific Test Files

Run individual test modules:

```bash
# Data validation tests
pytest tests/test_mock_data.py

# KPI calculation tests
pytest tests/test_kpis.py

# Alert engine tests
pytest tests/test_alerts.py
```

### Verbose Output

Show detailed test output:

```bash
pytest -v
```

### Run with Print Statements

Capture print output for debugging:

```bash
pytest -s
```

### Run Specific Test Function

```bash
pytest tests/test_kpis.py::test_pareto_suma_100
```

## Test Coverage

The test suite validates the following areas:

### 1. Mock Data Validation (`test_mock_data.py`)

Ensures the generated mock dataset meets business requirements:

- **Excel structure**: All required sheets exist (Ventas, Vendedores, Clientes, Productos, Objetivos)
- **Record counts**: 12 vendors, 180 customers, 60 products
- **Transaction volume**: Between 8,000-40,000 sales transactions
- **Customer status**: ~85% active, ~15% inactive
- **Data coherence**: 96%+ of sales match vendor-customer assignments
- **Seasonality**: December/January have 15%+ higher sales than average
- **Inactive customers**: No sales in last 60 days for inactive customers
- **Low-rotation products**: At least 8 products represent less than 5% of volume
- **Data quality**: No nulls in critical fields, positive amounts, valid date ranges

**Example test** (`test_mock_data.py:70-74`):
```python
def test_volumen_transacciones(data):
```

### 2. KPI Engine Tests (`test_kpis.py`)

Validates business logic calculations:

#### Pareto Analysis
- **Participation sums to 100%**: All percentages total 1.0
- **Category A covers 70-90%**: Pareto principle validation
- **No nulls**: All fields populated
- **Valid categories**: Only A, B, C categories
- **Monotonic cumulative**: Accumulated percentage increases monotonically

**Example test** (`test_kpis.py:91-98`):
```python
def test_pareto_suma_100():
    """Participation % sums exactly to 1.0 (scale 0-1)."""
    from src.pareto import calcular_pareto
    df = make_simple_df()
    grp = df.groupby("id_producto")["importe_neto"].sum().reset_index()
    result = calcular_pareto(grp, "importe_neto", "id_producto")
    total_pct = result["participacion_pct"].sum()
    assert abs(total_pct - 1.0) < 0.01
```

## Test Organization

### Test Fixtures

Reusable data fixtures for consistent testing:

```python
@pytest.fixture(scope="module")
def data():
    """Load Excel once for all tests."""
    if not os.path.exists(DATA_PATH):
        pytest.skip(f"Data file not found: {DATA_PATH}")
    xl = pd.ExcelFile(DATA_PATH)
    return {
        "ventas":     xl.parse("Ventas"),
        "vendedores": xl.parse("Vendedores"),
        "clientes":   xl.parse("Clientes"),
        "productos":  xl.parse("Productos"),
        "objetivos":  xl.parse("Objetivos"),
    }
```

### Test Helpers

Helper functions generate minimal DataFrames for unit tests:

```python
def make_simple_df(n: int = 500, seed: int = 42) -> pd.DataFrame:
    """Minimal sales DataFrame for pure unit tests."""
    np.random.seed(seed)
    return pd.DataFrame({
        "id_venta":      [f"T{i:04d}" for i in range(n)],
        "fecha":         pd.date_range("2024-01-01", periods=n),
        "id_vendedor":   np.random.choice(["V001", "V002", "V003"], n),
        "importe_neto":  np.random.uniform(5_000, 200_000, n),
    })
```

## Testing Best Practices

### 1. Run Tests Before Committing

Always run the full test suite before pushing changes:

```bash
pytest -v
```

### 2. Test-Driven Development

When adding new KPIs or alerts:
1. Write the test first
2. Implement the feature
3. Run tests to validate

### 3. Use Fixtures for Real Data

Real data tests automatically skip if mock data is not generated:

```python
if not os.path.exists(DATA_PATH):
    pytest.skip("Run: python data/mock/generate_mock_data.py")
```

### 4. Validate Edge Cases

Each test suite includes edge cases:
- Empty DataFrames
- Missing columns
- Null values
- Boundary conditions

## Continuous Integration

To set up CI/CD testing:

### GitHub Actions Example

```yaml
name: Tests

on: [push, pull_request]

jobs:
  test:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3
      - name: Set up Python
        uses: actions/setup-python@v4
        with:
          python-version: '3.9'
      - name: Install dependencies
        run: |
          pip install -r requirements.txt
      - name: Generate mock data
        run: |
          python data/mock/generate_mock_data.py
      - name: Run tests
        run: |
          pytest -v
```
