## Overview

Visor KPI Comercial is a Streamlit-based business intelligence dashboard for commercial KPI analysis. The application follows a modular architecture with clear separation between data processing, business logic, and presentation layers.

## Project Structure

The codebase is organized into the following directories:

```
visor_kpi/
├── app.py                    # Main entry point (Executive Summary page)
├── config.py                 # Global configuration and constants
├── requirements.txt          # Python dependencies
├── assets/                   # CSS and static files
├── components/              # Reusable UI components
│   ├── charts.py            # Chart rendering functions
│   ├── filters.py           # Sidebar filters
│   ├── kpi_card.py          # KPI card component
│   └── rankings.py          # Ranking tables
├── src/                     # Business logic layer
│   ├── data_loader.py       # Data loading and filtering
│   ├── kpi_engine.py        # KPI calculation engine
│   ├── alerts_engine.py     # Alert detection system
│   ├── pareto.py            # Pareto analysis
│   └── reports.py           # Report generation
├── pages/                   # Streamlit multi-page views
│   ├── 1_gerencia.py        # Management view
│   ├── 2_vendedores.py      # Sales team analysis
│   ├── 3_clientes.py        # Customer analysis
│   ├── 4_productos.py       # Product analysis
│   └── 5_alertas.py         # Alert center

└── tests/                   # Test suite
    ├── test_mock_data.py

    └── test_alerts.py
```

## Layer Separation

### Business Logic Layer (`src/`)

Contains pure Python functions for data processing and calculations:

- **`kpi_engine.py`**: Core KPI calculations (sales metrics, margins, coverage, rankings)
- **`alerts_engine.py`**: Business rules engine for detecting critical situations (ALT001-ALT007)
- **`pareto.py`**: Pareto analysis (ABC categorization)
- **`data_loader.py`**: Data loading, caching, and filtering
- **`reports.py`**: Report generation logic

### Presentation Layer (`components/`)

Reusable UI components for consistent design:

- **`kpi_card.py`**: KPI card with value, delta, and traffic light indicator
- **`charts.py`**: Chart factory functions (evolution, gauges, donuts, bars)
- **`filters.py`**: Sidebar filter controls with session state management
- **`rankings.py`**: Formatted ranking tables for top performers

### View Layer (`pages/`)

Streamlit pages that compose components and business logic:

- Executive Summary (`app.py`)
- Management Dashboard (gerencia)
- Sales Team Analysis (vendedores)
- Customer Analysis (clientes)
- Product Analysis (productos)
- Alert Center (alertas)

### Performance Thresholds

Business thresholds are configured in `config.py:38-49`:

```python
cumplimiento_ok:      1.00   # 100% → green
cumplimiento_warning: 0.80   # 80%  → orange
caida_critica:       -0.20   # -20% vs previous → alert
crecimiento_notable:  0.15   # +15% vs previous → highlight
pareto_acumulado:     0.80   # 80% volume = A customers
pareto_B:             0.95   # 80-95% = B, >95% = C
```

### Typography & Formatting

All number formatting functions are centralized in `config.py:66-168`:

- **`fmt_moneda(valor)`**: Currency formatting (e.g., `$1.284.500`)
- **`fmt_pct(valor)`**: Percentage formatting (e.g., `84.2%`)
- **`fmt_numero(valor)`**: Number formatting with thousand separators
- **`fmt_delta(valor)`**: Delta with arrows (▲ +8.2% / ▼ -3.1%)
- **`fmt_delta_html(valor)`**: Delta with HTML color coding

## Data Flow

1. **Data Loading**: Excel file loaded via `data_loader.py` with caching
2. **Filtering**: User selections applied via `get_filtered_data()`
3. **KPI Calculation**: Business logic in `kpi_engine.py` processes filtered data
4. **Alert Detection**: `alerts_engine.py` evaluates business rules
5. **Visualization**: Components render KPIs and charts
6. **Rendering**: Streamlit displays the complete dashboard

## State Management

Session state is initialized in `components/filters.py` to maintain user selections across page navigation:
