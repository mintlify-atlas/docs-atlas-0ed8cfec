## Overview

The `kpi_margen()` function calculates gross margin metrics for a sales period, including total margin amount, average margin percentage, and period-over-period comparisons.

<Note>
  Margin analysis is critical for understanding **profitability**, not just revenue. A high-volume seller with low margins may be less valuable than a moderate seller with healthy margins.
</Note>

## Why It Matters

Margin KPIs answer essential profitability questions:

- **Is the business making money?** Revenue without margin visibility can be misleading
- **Which segments are most profitable?** Compare margin % across sellers, products, or channels
- **Are margins improving or eroding?** Period comparisons reveal pricing and cost trends
- **Who needs coaching?** Identify sellers discounting excessively

## Function Signature

```python kpi_engine.py
def kpi_margen(
    df: pd.DataFrame,
    df_anterior: pd.DataFrame | None = None,
) -> dict:
    """
    KPIs de margen bruto.
    """
```

## Parameters

<ParamField path="df" type="pd.DataFrame" required>
  Sales DataFrame for the current period. Must contain columns:
  - `margen_neto`: Net margin amount per transaction
  - `importe_neto`: Net sales amount per transaction
  
  If `margen_neto` column is missing, returns zero values.
</ParamField>

<ParamField path="df_anterior" type="pd.DataFrame | None" default="None">
  Sales DataFrame from the previous period for margin trend analysis. Should contain:
  - `margen_neto`: Net margin amount
  
  When provided, calculates `margen_vs_anterior` (% change in margin).
</ParamField>

## Return Value

Returns a dictionary with the following structure:

| Key | Type | Description |
|-----|------|-------------|
| `margen_bruto_total` | `float` | Total gross margin in currency units |
| `margen_pct_promedio` | `float` | Average margin as a percentage (e.g., 0.25 = 25% margin) |
| `margen_vs_anterior` | `float \| None` | % change in total margin vs. previous period |

<Info>
  `margen_pct_promedio` is calculated as total margin divided by total sales, representing the **blended margin rate** for the entire period.
</Info>

```python Margin Alert System
from src.kpi_engine import kpi_margen

# Define margin thresholds
MARGIN_MIN_THRESHOLD = 0.20  # 20%
MARGIN_TARGET = 0.25         # 25%

current_margin = margin['margen_pct_promedio']

if current_margin < MARGIN_MIN_THRESHOLD:

    print(f"⚠️  WARNING: Margin at {current_margin:.1%} is below target")

    print(f"✅ Margin at {current_margin:.1%} meets target")

if margin['margen_vs_anterior'] is not None:
    if margin['margen_vs_anterior'] < -0.05:  # 5% decline
        print(f"\n📉 WARNING: Margin declined {margin['margen_vs_anterior']:.1%} vs previous period")
```

## Business Logic

The function implements the following calculation sequence:

### 1. Data Validation
```python
if df.empty or "margen_neto" not in df.columns:
    return {"margen_bruto_total": 0, "margen_pct_promedio": 0, "margen_vs_anterior": None}
```

### 2. Total Margin Calculation
```python
margen_total = float(df["margen_neto"].sum())
importe_total = float(df["importe_neto"].sum())
```

### 3. Margin Percentage
Calculates the blended margin rate for the period:
```python
margen_pct = margen_total / importe_total if importe_total > 0 else 0.0
```

<Warning>
  This is **not** an average of individual transaction margins, but rather total margin divided by total sales. This gives more weight to high-volume transactions.
</Warning>

### 4. Period Comparison
```python
margen_vs_ant = None
if df_anterior is not None and not df_anterior.empty and "margen_neto" in df_anterior.columns:
    margen_ant = float(df_anterior["margen_neto"].sum())
    if margen_ant > 0:
        margen_vs_ant = (margen_total - margen_ant) / margen_ant
```
