## Overview

The `kpi_cobertura_clientes()` function analyzes customer portfolio health by calculating coverage rates, identifying churned customers, tracking new acquisitions, and monitoring reactivations.

<Note>
  Coverage metrics answer the critical question: **"Are we actively serving our customer base, or are we leaving money on the table?"**
</Note>

## Why It Matters

Customer coverage is a leading indicator of sales health:

- **Low coverage** signals underutilized territory potential or execution problems
- **Churned customers** represent revenue at risk that requires immediate action
- **New customer acquisition** measures market expansion effectiveness
- **Reactivations** show success in winning back dormant accounts

Unlike lagging indicators (revenue, margin), coverage metrics provide **early warning signals** before problems impact the bottom line.

## Function Signature

```python kpi_engine.py
def kpi_cobertura_clientes(
    df: pd.DataFrame,
    clientes_df: pd.DataFrame,
    df_anterior: pd.DataFrame | None = None,
) -> dict:
    """
    KPIs de cobertura y estado de la cartera de clientes.
    """
```

## Parameters

<ParamField path="df" type="pd.DataFrame" required>
  Sales transactions for the current period. Must contain:
  - `id_cliente`: Customer identifier for each transaction
  
  Used to identify which customers purchased in the current period.
</ParamField>

<ParamField path="clientes_df" type="pd.DataFrame" required>
  Master customer list with columns:
  - `id_cliente`: Unique customer identifier
  - `activo`: Boolean flag indicating if customer is active in the system
  
  This represents the **total addressable customer base**.
</ParamField>

<ParamField path="df_anterior" type="pd.DataFrame | None" default="None">
  Sales transactions from the previous period. Required to calculate:
  - Churned customers (bought before but not now)
  - New customers (first purchase in current period)
  - Reactivated customers (returned after absence)
</ParamField>

## Return Value

Returns a dictionary with comprehensive coverage metrics:

| Key | Type | Description |
|-----|------|-------------|
| `clientes_activos_periodo` | `int` | Number of active customers who purchased this period |
| `clientes_total` | `int` | Total active customers in the portfolio |
| `cobertura_pct` | `float` | Coverage rate (active buyers / total active customers) |
| `clientes_perdidos` | `int` | Customers who bought last period but not this period |
| `clientes_nuevos` | `int` | Customers making their first purchase this period |
| `clientes_recuperados` | `int` | Previously inactive customers who returned |

```python Coverage Alert System
from src.kpi_engine import kpi_cobertura_clientes

# Define thresholds
COVERAGE_MIN = 0.60  # 60% minimum
COVERAGE_TARGET = 0.75  # 75% target
CHURN_ALERT_THRESHOLD = 10  # Alert if 10+ customers churned

coverage = kpi_cobertura_clientes(
    df=df_marzo,
    clientes_df=df_clientes,
    df_anterior=df_febrero
)

# Coverage alerts
if coverage['cobertura_pct'] < COVERAGE_MIN:
    print(f"🚨 CRITICAL: Coverage at {coverage['cobertura_pct']:.1%} is below minimum")
    print(f"   {coverage['clientes_total'] - coverage['clientes_activos_periodo']} customers not served")
elif coverage['cobertura_pct'] < COVERAGE_TARGET:
    print(f"⚠️  WARNING: Coverage at {coverage['cobertura_pct']:.1%} below target")
    gap = int((COVERAGE_TARGET - coverage['cobertura_pct']) * coverage['clientes_total'])
    print(f"   Need {gap} more active customers to reach target")
else:
    print(f"✅ Coverage at {coverage['cobertura_pct']:.1%} meets target")

# Churn alerts
if coverage['clientes_perdidos'] >= CHURN_ALERT_THRESHOLD:
    print(f"\n🚨 CHURN ALERT: {coverage['clientes_perdidos']} customers lost")
    print(f"   Action: Review churned customer list and assign recovery tasks")

# Positive signals
if coverage['clientes_nuevos'] > coverage['clientes_perdidos']:
    print(f"\n📈 Positive: New customers ({coverage['clientes_nuevos']}) exceed churn")
```

from src.kpi_engine import kpi_cobertura_clientes
import pandas as pd

# Calculate coverage for last 3 periods
periods = [
    {'name': 'March', 'start': date(2026, 3, 1), 'end': date(2026, 3, 31)},

df_trend = pd.DataFrame(results)

## Business Logic

The function implements sophisticated customer lifecycle tracking:

### 1. Identify Total Portfolio
```python
clientes_activos_total = clientes_df[clientes_df["activo"] == True]["id_cliente"].tolist()
n_total = len(clientes_activos_total)
```

### 2. Calculate Current Period Buyers
```python
compraron = set(df["id_cliente"].unique()) if not df.empty else set()
n_activos_periodo = len(compraron & set(clientes_activos_total))
```

### 3. Coverage Rate
```python
cobertura = n_activos_periodo / n_total if n_total > 0 else 0.0
```

### 4. Churn Detection
Customers who bought last period but not this period:
```python
compraron_anterior = set(df_anterior["id_cliente"].unique())
n_perdidos = len(compraron_anterior - compraron)
```

### 5. New Customer Identification
Customers who never purchased before:
```python
todos_anteriores = set(df_anterior["id_cliente"].unique())
n_nuevos = len(compraron - todos_anteriores)
```

## Key Insights

<AccordionGroup>
  <Accordion title="What is a 'good' coverage rate?">
    Coverage benchmarks vary by industry and sales model:
    
    - **60-70%**: Minimum acceptable for most B2B distribution
    - **75-85%**: Healthy coverage indicating good execution
    - **>85%**: Excellent coverage (may indicate market saturation)
    
    Low coverage (less than 60%) usually indicates:
    - Insufficient sales visit frequency
    - Territory too large for seller capacity
    - Inactive customers not being removed from portfolio
  </Accordion>
  
  <Accordion title="When is churn vs. normal customer behavior?">
    Not all "churned" customers are lost:
    
    - **B2B**: Many customers don't buy every period (seasonal, project-based)
    - **Threshold**: 60+ days without purchase may indicate true churn
    - **Context matters**: Compare to customer's historical frequency
    
    The `kpi_clientes()` function provides `dias_desde_ultima_compra` for deeper analysis.
  </Accordion>
  
  <Accordion title="How to interpret reactivations?">
    Reactivated customers indicate:
    
    - **Successful win-back campaigns**
    - **Seasonal purchasing patterns**
    - **Improved seller territory management**
    
    High reactivation rates may also signal:
    - Customer base was incorrectly marked as churned
    - Need to adjust churn definition timeframe
  </Accordion>
</AccordionGroup>
